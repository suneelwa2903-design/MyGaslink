## 7. Known risks to watch for during testing

1. **GST UI on `gstMode=DISABLED` Bhargava tenant** — credit-note / debit-note buttons should NOT appear. If they do, it's a regression of WI-004. Test 2.6 area.
2. **Driver phone in customer portal** — only visible during `pending_dispatch` / `pending_delivery`. If visible on a delivered order or before assignment, it's a Test 6.3 bug.
3. **Offline delivery sync** — the queue lives in expo-secure-store. If the phone is fully killed (force-quit + reboot) the queue should still survive. If it doesn't survive, the storage key handling has a bug. Test 5.3 covers the basic flow but not full reboot.
4. **Postgres advisory lock during dev migrations** — multiple worktrees on the shared local DB can lock each other out. The `LOCAL-DEV-STARTUP.md` troubleshooting section documents the kill-stale-connections workaround.
5. **Decimal money serialization** — every API JSON response now goes through `decimalsToNumbers` before serializing. If any UI shows a string-formatted number (`"1234.5600"`) instead of a number, a mapper is missing the wrap. Tests cover the API side; UI smoke would catch this.
6. **Telugu language switcher** — turning Telugu on shows English fallback strings on un-extracted pages. That's expected, not a bug. Communicate to early customers if they switch.
7. **Idempotent delivery confirm** — duplicate retry with same quantities returns 200, with different quantities returns 409. If a duplicate ever creates a second invoice / inventory event, that's a regression of WI-027. Best caught by the API test suite, also visible in Test 5.3 if the same delivery ends up double-recorded on the admin side.
